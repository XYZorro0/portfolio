# Portfolio AI Chatbot

A chat widget for niketgupta.com that answers visitor questions about Niket using retrieval-augmented generation (RAG) over his resume, project write-ups, and bio — powered by a locally hosted Ollama model on the home laptop (Lenovo IdeaPad L340: i7-9750H, GTX 1650 4GB, 16GB RAM). The bot answers **only** questions about Niket and politely redirects everything else.

```
Visitor on niketgupta.com (Vercel)
        │  fetch POST /api/chat  (X-API-Key)
        ▼
Cloudflare Tunnel  →  chat-api.niketgupta.com
        │
        ▼
Backend API (this folder, Express on 127.0.0.1:3001)
   API key check → rate limit → embed question → cosine top-k
   over data/vector-store.json → prompt with retrieved context
        │
        ▼
Ollama (127.0.0.1:11434, never exposed)  →  llama3.2:3b
```

## Layout

| Path | What it is |
|---|---|
| `server.js` + `lib/` | API server: auth, rate limiting, CORS, retrieval, Ollama proxy |
| `ingest/ingest.js` | Chunks + embeds `ingest/docs/*.md` into the vector store |
| `ingest/docs/` | The knowledge base (resume, bio, projects, certifications) |
| `data/vector-store.json` | Generated by ingestion (gitignored — rebuild with `npm run ingest`) |
| `deploy/cloudflared-config.example.yml` | Cloudflare Tunnel ingress config |
| `test/` | Mock Ollama + smoke test (`npm test`, no GPU needed) |
| `../frontend/widget/` | Embeddable chat widget + local demo page |

## 1. Install Ollama and pull the models

```bash
# Linux
curl -fsSL https://ollama.com/install.sh | sh
# Windows/macOS: install from https://ollama.com/download

ollama pull llama3.2:3b        # primary chat model (~2.0 GB, fits the GTX 1650)
ollama pull phi4-mini          # optional alternate to compare
ollama pull nomic-embed-text   # embedding model (~274 MB)
```

Ollama listens on `127.0.0.1:11434` by default — leave it that way. Don't set
`OLLAMA_HOST=0.0.0.0`, and never add port 11434 to the tunnel config.

## 2. Configure and start the backend

```bash
cd backend
npm install
cp .env.example .env
# generate a widget key and put it in .env as CHATBOT_API_KEY:
node -e "console.log(crypto.randomBytes(24).toString('base64url'))"
```

Every knob (models, rate limits, CORS origins, similarity threshold) is
documented in `.env.example`.

## 3. Build the knowledge base

With Ollama running:

```bash
npm run ingest
```

This chunks every markdown file in `ingest/docs/`, embeds the chunks with
`nomic-embed-text`, and writes `data/vector-store.json`. Re-run it whenever
you edit the docs (then restart the server, which loads the store at startup).
To add content, just drop more `.md` files into `ingest/docs/`.

## 4. Run and test with curl

```bash
npm start          # or: npm run dev  (auto-restarts on file changes)

curl http://127.0.0.1:3001/api/health

curl -X POST http://127.0.0.1:3001/api/chat \
  -H 'Content-Type: application/json' \
  -H 'X-API-Key: YOUR_KEY' \
  -d '{"message": "What projects has Niket built?"}'
```

Expected response shape:

```json
{ "answer": "...", "offTopic": false, "sources": ["Dual-Pipeline Lung Cancer Detection System"] }
```

Off-topic questions skip the LLM entirely (retrieval similarity below
`MIN_SIMILARITY`) and return a canned redirect with `"offTopic": true` — zero
GPU time spent on abuse.

To try the alternate model: `ollama pull phi4-mini`, set
`CHAT_MODEL=phi4-mini` in `.env`, restart, and compare answers.

**No GPU handy?** `npm test` runs the whole pipeline (ingest → server → auth →
retrieval → rate limit) against a mock Ollama on port 11435.

## 5. Expose it with Cloudflare Tunnel

Never port-forward the laptop. The tunnel makes outbound-only connections to
Cloudflare, so nothing on the home network accepts inbound traffic.

```bash
# 1. Install cloudflared: https://developers.cloudflare.com/cloudflare-one/connections/connect-networks/downloads/
# 2. Authenticate against the niketgupta.com zone (opens a browser):
cloudflared tunnel login

# 3. Create the tunnel and note the ID it prints:
cloudflared tunnel create portfolio-chatbot

# 4. Copy deploy/cloudflared-config.example.yml to ~/.cloudflared/config.yml
#    and fill in the tunnel ID.

# 5. Point DNS at the tunnel (creates the CNAME in Cloudflare):
cloudflared tunnel route dns portfolio-chatbot chat-api.niketgupta.com

# 6. Test drive, then install as a service so it survives reboots:
cloudflared tunnel run portfolio-chatbot
sudo cloudflared service install     # Linux (systemd)
# Windows: run an elevated prompt ->  cloudflared service install
```

Verify from anywhere: `curl https://chat-api.niketgupta.com/api/health`

Also make the backend itself survive reboots — e.g. a systemd unit for
`node server.js` on Linux, or Task Scheduler / NSSM on Windows, plus
`ollama serve` (the installers set Ollama up as a service already).

## 6. Embed the widget on the site

The site's CSP only allows scripts from its own origin, so serve the widget
from the Next.js app rather than a third-party host:

1. Copy `frontend/widget/chat-widget.js` into the site's `public/` folder.
2. Add the tag to the site layout (any plain `<script>` slot works):

   ```html
   <script src="/chat-widget.js" defer
           data-api-url="https://chat-api.niketgupta.com"
           data-api-key="YOUR_WIDGET_KEY"></script>
   ```

3. **CSP:** add the API origin to `connect-src` in `next.config.ts`, or the
   browser will block the widget's fetch:

   ```
   connect-src 'self' blob: data: https://ruucm.github.io https://chat-api.niketgupta.com
   ```

To preview locally first: `npm run dev` here, then open
`frontend/widget/demo.html` via any static server (the demo page points at
`http://127.0.0.1:3001`; add the demo's origin, e.g.
`http://127.0.0.1:8080`, to `ALLOWED_ORIGINS` in `.env`).

## Security model

- **Ollama is never reachable from the internet** — it binds to localhost and
  the tunnel only routes to the backend port.
- **The backend binds to 127.0.0.1** — only cloudflared can reach it.
- **Every request needs the API key** (`X-API-Key`, compared timing-safe).
  The key ships inside the public widget JS, so treat it as abuse friction,
  not a secret; rotating it is a one-line change in `.env` + the script tag.
- **Rate limiting**: per-IP fixed window (default 10 req/min) using the
  visitor IP cloudflared forwards, plus a single-file GPU queue — one
  generation at a time, a short wait queue, then 429. A traffic spike
  degrades into "try again in a few seconds", not a melted GTX 1650.
- **CORS locked** to `https://niketgupta.com` / `https://www.niketgupta.com`;
  other origins get a 403 and no CORS headers.
- **Prompt scope**: the system prompt restricts answers to the retrieved
  context about Niket, and the retrieval threshold rejects off-topic
  questions before they ever reach the model.
